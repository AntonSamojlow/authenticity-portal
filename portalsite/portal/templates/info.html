About this portal

This is a prototype of a web site/server that

  • serves selected prediction models to a general audience
  • presents the models within a specific topic (use case), providing additional information and context

It is named authenticity-portal to indicate one possible field of application: A platform where one can upload some (measurement) data of a food to verify/predict certain properties, like authenticity of the geographical organic. The prediction models are curated and validated by trusted institutions.

You are welcome to use one of two pre-made users accounts to explore the functionality of this site:

Usage description

The portal is designed to accomodate two different work flows (and user types):

General userscan obtain information about their dataset from validated models, in particular:

  • a quantitative prediction, like the nutrient content of a sample
  • a classification, like the authenticity of a sample (likelihood of class member ship)

This user typically interactes by accessing a specific topic section, over the entry page (home) or by directly accessing the relevant topic, like the iris section. Here, selected models are listed and available to use for a prediction via the 'Use for a prediction button' in the 'Actions' column. This leads to a form where the user may submit his dataset.

The data manager or scientist curates the available validated models. This involves

  • upload of new measurment data: see measurement overview
  • generation of new models: see model overview
  • training of models: select a specific model from the model overview and choose from compativle measruements to train on
  • declaring a model as valid for use by general users: having a specific model select sel√¶ected its 'Ready for predictions' status can be changed

This user typically has advanced access to the site and also operates in the data overview sections for all models and measurements. In contrast, the topic pages only present a subset, since models and measurements each have a group tag used to filter (the iris section for example shows only objects belonging to at least one group with 'iris' in its name ). Moreover, selected users have staff privileges, which enable have full control over data tables behind this site via djangos Admin portal.

Topics

Three topics are presented: Iris, Salmon and Vanilla. The first is built around the common iris set. The other two are (currently) just dummy pages. Maybe you can fill them with meaningful data and models ;) ?

Models

Currently, two model types are implemented: Linear regression as a simple example of a model capable of quantitative predictions. And SIMCA as an example of a classification model (predicting the likelihood of class memebership). The latter may not be well-known in all fields - it was ported by me to python from the R package 'mdatools' for use in this portal, see also this reference.

More model types can be easily added in python. For other languages, additional work (or a different platform than heroku) might be required.
